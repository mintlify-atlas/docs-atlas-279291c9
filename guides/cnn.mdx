Convolutional networks produce 3D feature maps — channels, height, and width — before the final classification head. The standard approach flattens these maps into a 1D vector and feeds them into `nn.Linear`, discarding the spatial structure that the convolutions worked hard to extract.

NdLinear preserves that structure. Instead of flattening `(C, H, W)` into a single long vector, it operates directly on the 3D tensor, learning separate projections along each dimension.

## The HyperVision model

The following model classifies CIFAR-10 images (`3 × 32 × 32`). After two pooling stages, the feature maps are `64 × 8 × 8`. NdLinear replaces the flatten-then-linear step.

```python
import math
import torch.nn as nn
from ndlinear import NdLinear

class HyperVision(nn.Module):
    def __init__(self, input_shape, hidden_size, num_classes=10):
        super(HyperVision, self).__init__()
        self.hyper_conv1 = nn.Conv2d(3, 32, kernel_size=3, padding=1)
        self.hyper_bn1 = nn.BatchNorm2d(32)
        self.hyper_conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.hyper_bn2 = nn.BatchNorm2d(64)
        self.hyper_dropout2d = nn.Dropout2d(0.2)

        # Operates on (64, 8, 8) feature maps directly — no flatten needed
        self.hyper_ndlinear = NdLinear((64, 8, 8), hidden_size)

        final_dim = math.prod(hidden_size)
        self.hyper_fc_out = nn.Linear(final_dim, num_classes)
        self.hyper_dropout = nn.Dropout(0.3)
        self.hyper_relu = nn.ReLU()
        self.hyper_pool = nn.MaxPool2d(2, 2)

    def forward(self, x):
        x = self.hyper_pool(self.hyper_relu(self.hyper_bn1(self.hyper_conv1(x))))
        x = self.hyper_dropout2d(x)
        x = self.hyper_pool(self.hyper_relu(self.hyper_bn2(self.hyper_conv2(x))))
        x = self.hyper_dropout2d(x)

        # (batch, 64, 8, 8) → (batch, *hidden_size)
        x = self.hyper_ndlinear(x)
        x = x.view(x.shape[0], -1)
        x = self.hyper_dropout(self.hyper_fc_out(self.hyper_relu(x)))
        return x
```

Instantiate the model:

```python
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model = HyperVision(input_shape=(3, 32, 32), hidden_size=(64, 8, 8)).to(device)
```

The `hidden_size=(64, 8, 8)` argument tells NdLinear to project the `(64, 8, 8)` feature map to another `(64, 8, 8)` tensor. After flattening that output, the final `nn.Linear` maps `64 × 8 × 8 = 4096` features to 10 classes.

## Why NdLinear works well here

After two `MaxPool2d(2, 2)` operations on a `32 × 32` input, the spatial resolution is `8 × 8`. A standard `nn.Linear` would need to treat all `64 × 8 × 8 = 4096` activations as interchangeable scalars.

NdLinear instead applies three separate projections — one along the channel dimension, one along height, and one along width — then combines them. The spatial relationships that `Conv2d` learned are preserved through the transition to the classification head.

## Parameter count comparison

For `hidden_size=(64, 8, 8)` and 10 output classes:

| Layer | Approach | Parameters |
|-------|----------|------------|
| Feature projection | `nn.Linear(4096, 4096)` | 16,777,216 |
| Feature projection | `NdLinear((64,8,8), (64,8,8))` | 64×64 + 8×8 + 8×8 = **4,224** |
| Output head | `nn.Linear(4096, 10)` | 40,970 |

<Note>
The NdLinear projection uses roughly **4,000× fewer parameters** than a comparable `nn.Linear` for this feature map size. See [Parameter efficiency](/concepts/parameter-efficiency) for the full breakdown.
</Note>
