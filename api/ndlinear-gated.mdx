---
title: "NdLinearGated"
description: "An NdLinear layer with per-dimension gating to selectively apply transformations."
---

`NdLinearGated` extends [`NdLinear`](/api/ndlinear) with a learned gating mechanism that controls whether each dimension's transformation is applied. Gates are computed from the input itself and can be soft (continuous blending) or hard (binary switching), and can be applied to all dimensions or a learned subset.

This allows the model to skip projections that are not useful for a given input, reducing compute and improving interpretability.

## Constructor

```python
class NdLinearGated(nn.Module):
    def __init__(
        self,
        input_dims: tuple,
        hidden_size: tuple,
        transform_outer: bool = True,
        gating_mode: Literal["soft", "hard"] = "soft",
        gating_hidden_dim: int = 16,
        gated_modes: Literal["all", "first", "topk"] = "all",
    ) -> None
```

<ParamField path="input_dims" type="tuple" required>
  Shape of the input tensor, excluding the batch dimension. For example, an image of shape `[batch, 28, 28, 3]` has `input_dims=(28, 28, 3)`.
</ParamField>

<ParamField path="hidden_size" type="tuple" required>
  Target output dimensions after transformation. Must have the same length as `input_dims`.
</ParamField>

<ParamField path="transform_outer" type="boolean" default="True">
  If `True`, transformations are applied from the outermost dimension inward.
</ParamField>

<ParamField path="gating_mode" type="soft | hard" default="soft">
  Type of gating mechanism applied at each gated dimension.

  - `"soft"` — Gate is a continuous value computed as `sigmoid(linear(x))`. The output blends the transformed and identity representations: `output = gate * transformed + (1 - gate) * identity`. The model can learn to partially apply a projection.
  - `"hard"` — Gate is binarized: if `gate > 0.5`, the transformed output is used; otherwise the identity (untransformed) output is used. This produces sparser, more interpretable routing but is not differentiable through the gating decision.
</ParamField>

<ParamField path="gating_hidden_dim" type="integer" default="16">
  Hidden dimension of the small MLP used to compute each gate from the input. Larger values increase the gating network's capacity at a minor cost to parameter count.
</ParamField>

<ParamField path="gated_modes" type="all | first | topk" default="all">
  Which dimensions receive a gate.

  - `"all"` — Every dimension has its own gate. Full gating overhead; maximum flexibility.
  - `"first"` — Only the first dimension (outermost) is gated. All other dimensions are always transformed. Lowest overhead.
  - `"topk"` — Gates are applied to the top-2 dimensions ranked by standard deviation, computed lazily on the first forward pass. This focuses compute on the most variable dimensions and reuses the same selection for all subsequent batches.
</ParamField>

<Note>
  When `gated_modes="topk"`, the set of gated dimensions (`topk_modes`) is computed on the first forward pass and cached. It is not updated during training. If the input distribution shifts significantly after the first batch, the cached selection may be suboptimal.
</Note>

## Gating behavior

### Soft gating

```python
gate = torch.sigmoid(gate_network(x))          # values in (0, 1)
output = gate * transformed + (1 - gate) * identity
```

The projection is applied proportionally. A gate near `1.0` uses the full transformation; a gate near `0.0` passes through the identity.

### Hard gating

```python
if gate > 0.5:
    output = transformed
else:
    output = identity
```

The projection is either fully applied or fully skipped. Hard gating produces binary routing decisions and can reduce inference compute when gates are consistently low for certain dimensions.

## Gating configurations

### Soft gating on all dimensions

```python
from ndlinear import NdLinearGated

layer = NdLinearGated(
    input_dims=(28, 28, 3),
    hidden_size=(64, 64, 6),
    gating_mode="soft",
    gated_modes="all",
)
```

Every dimension has a learned continuous gate. Most flexible configuration. Suitable when you have no prior knowledge about which dimensions matter.

### Hard gating on first dimension only

```python
from ndlinear import NdLinearGated

layer = NdLinearGated(
    input_dims=(28, 28, 3),
    hidden_size=(64, 64, 6),
    gating_mode="hard",
    gated_modes="first",
)
```

Only the first (height) dimension is gated, and gating is binary. The remaining dimensions are always projected. Lowest parameter and compute overhead among gated configurations.

### Soft gating on top-2 dimensions by variance (recommended)

```python
from ndlinear import NdLinearGated

optimal_gated = NdLinearGated(
    input_dims=(28, 28, 3),
    hidden_size=(64, 64, 6),
    gating_mode="soft",
    gated_modes="topk",
    gating_hidden_dim=16,
)
```

This is the recommended configuration. It applies soft gates only to the two dimensions with the highest standard deviation on the first batch, leaving low-variance dimensions ungated.

**Why it works well:**

- **~0.5–0.7% accuracy improvement** over ungated `NdLinear` on typical vision benchmarks.
- **50–75% compute reduction** by skipping projections for dimensions where the gate consistently learns to pass through the identity.
- **Stable training** because soft gates are differentiable and the gating network is small.
- **Better interpretability** — the model explicitly signals which dimensions it finds useful to transform.

## Full example

```python
import torch
from ndlinear import NdLinearGated

layer = NdLinearGated(
    input_dims=(28, 28, 3),
    hidden_size=(64, 64, 6),
    gating_mode="soft",
    gated_modes="topk",
    gating_hidden_dim=16,
)

batch = torch.randn(16, 28, 28, 3)

# First forward pass: topk_modes is computed and cached
output = layer(batch)
print(output.shape)  # torch.Size([16, 64, 64, 6])

# Subsequent passes reuse the cached topk_modes
output2 = layer(batch)
print(output2.shape)  # torch.Size([16, 64, 64, 6])
```

<Tip>
  Start with `gating_mode="soft"` and `gated_modes="topk"`. Switch to `gated_modes="all"` only if you observe that gating is consistently low across all dimensions, which may indicate your input has high variance everywhere.
</Tip>
