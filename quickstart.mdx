This guide walks you through creating an NdLinear layer, running a forward pass, verifying output shapes, and comparing parameter counts against `nn.Linear`.

<Steps>
  <Step title="Install the package">
    If you haven't installed NdLinear yet, see the [installation guide](/installation). The minimal install is:

    ```bash
    pip install ndlinear
    ```
  </Step>

  <Step title="Create a layer and run a forward pass">
    The most common use case is transforming a batch of multi-dimensional inputs — for example, a batch of images with shape `(batch, height, width, channels)`.

    ```python
    import torch
    from ndlinear import NdLinear

    # Batch of 32 grayscale images: (batch, height, width, channels)
    x = torch.randn(32, 28, 28, 3)

    # input_dims and hidden_size must have the same length
    layer = NdLinear(input_dims=(28, 28, 3), hidden_size=(64, 64, 6))

    output = layer(x)
    ```

    <Tip>
      `input_dims` describes the shape of a single sample (excluding the batch dimension). `hidden_size` is the desired output shape for that sample, with one value per dimension.
    </Tip>
  </Step>

  <Step title="Verify the output shape">
    The output preserves the number of dimensions, with each dimension size mapped to the corresponding value in `hidden_size`:

    ```python
    print(output.shape)  # torch.Size([32, 64, 64, 6])
    assert output.shape == (32, 64, 64, 6)
    ```

    The batch dimension passes through unchanged. Only the dimensions listed in `input_dims` are transformed.
  </Step>

  <Step title="Compare parameter counts">
    One of the main benefits of NdLinear is parameter efficiency. Compare it against a naive `nn.Linear` that projects the same flattened input to the same flattened output:

    ```python
    import torch.nn as nn

    # Equivalent nn.Linear: flattens 28*28*3=2352 -> 64*64*6=24576
    linear = nn.Linear(28 * 28 * 3, 64 * 64 * 6)

    def count_params(m):
        return sum(p.numel() for p in m.parameters())

    print(f"nn.Linear params:  {count_params(linear):,}")
    print(f"NdLinear params:   {count_params(layer):,}")
    ```

    NdLinear applies one small linear per dimension rather than one large matrix, so the parameter count scales additively rather than multiplicatively.
  </Step>
</Steps>

## 1D edge case

When `input_dims` and `hidden_size` each have a single element, NdLinear behaves identically to `nn.Linear`:

```python
layer_1d = NdLinear(input_dims=(32,), hidden_size=(64,))
x = torch.randn(16, 32)
print(layer_1d(x).shape)  # torch.Size([16, 64])
```

This makes it safe to use NdLinear uniformly throughout a codebase without special-casing 1D projections.

## Using NdLinear in a transformer

To use NdLinear as a token-wise projection inside a transformer, reshape the sequence so each token occupies a separate batch element:

```python
# Input: (batch, seq_len, token_dim) = (32, 28, 28)
x = torch.randn(32, 28, 28)

# Treat each token as a (token_dim, 1) sample
x = x.reshape(-1, 28, 1)              # (32*28, 28, 1) = (896, 28, 1)

layer = NdLinear(input_dims=(28, 1), hidden_size=(32, 1))
out = layer(x)                        # (896, 32, 1)

out = out.reshape(32, 28, -1)         # (32, 28, 32)
print(out.shape)                      # torch.Size([32, 28, 32])
```

<Tip>
  Keep the trailing singleton dimension (`1`) in place when adapting sequences. It acts as a placeholder that lets NdLinear apply the transformation without changing the number of dimensions.
</Tip>
